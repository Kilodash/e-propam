# Project background

## Why

Aplikasi ini dibangun untuk memonitoring pengaduan masyarakat dan tindaklanjutnya di Bidpropam Polda Jawa Barat. Aplikasi asli (Gajamada Propam - gajamada-propam.polri.go.id) memiliki banyak kekurangan dalam hal UI/UX, alur kerja (flow), prosedur, dan fitur. Aplikasi ini memperbaiki kelemahan tersebut melalui sinkronisasi dua arah dengan sistem sumber.

## Goals

- Menyediakan interface yang lebih baik untuk monitoring pengaduan masyarakat
- Memperbaiki alur kerja (workflow) penerimaan, disposisi, distribusi, penyelidikan, dan tindaklanjut surat pengaduan
- Menyediakan pelacakan status pengaduan yang transparan di setiap level
- Sinkronisasi data dua arah dengan Gajamada Propam sebagai sumber data utama
- Mendukung seluruh unit kerja di lingkungan Polda Jabar

## Non-goals

- Pengaduan di luar wilayah Polda Jawa Barat
- Akses untuk masyarakat umum (aplikasi ini untuk internal Polda Jabar)

## Target users

Seluruh pengguna adalah internal Polda Jawa Barat dengan peran hierarkis:

| Peran | Unit | Tanggung Jawab |
|-------|------|----------------|
| Admin | - | Manajemen sistem |
| Kasubbag Yanduan | Operator Subbag Yanduan | Menerima surat tingkat Polda |
| Kabid Propam | - | Menerima surat, memberikan disposisi dan distribusi |
| Kasubbid Paminal | Unit Subbid Paminal | Menerima surat, penyelidikan |
| Kasubbid Provos | Unit Subbid Provos | Menerima surat, proses pelanggaran disiplin |
| Kasubbid Wabprof | Unit Subbid Wabprof | Menerima surat, proses pelanggaran kode etik profesi Polri |
| Kasubbag Rehabpers | Unit Subbag Rehabpers | Menerima surat, proses pemutihan/rekomendasi |
| Seksipropam Polres | Polres/Polresta/Polrestabes, Satbrimob | Menerima surat, proses di tingkat Polres |
| Wassidik | Ditreskrimum/Sus/Narkoba/Siber/PPA/PPO | Menerima pelimpahan terkait proses penanganan pidana |
